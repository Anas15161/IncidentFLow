Architecture & Modèle de Données

Comprenez l'organisation de l'infrastructure Docker-Compose, la gestion des identités Keycloak et les schémas de la base de données PostgreSQL.

Schéma des Relations & Cascades JPA

Les entités Hibernate mappent le schéma SQL PostgreSQL en appliquant des règles d'intégrité strictes :

Entité Origine Relation Cible Règle Cascade & Notes
Workflow OneToMany WorkflowState ALL, orphanRemoval=true (Supprimer un workflow efface tous ses états)
Workflow OneToMany WorkflowTransition ALL, orphanRemoval=true (Supprimer un workflow efface toutes ses transitions)
Incident OneToMany Comment ALL, orphanRemoval=true (Effacer l'incident supprime son fil de discussion)
Incident OneToMany Attachment ALL, orphanRemoval=true (Les pièces jointes sont nettoyées de la base)
Incident ManyToOne Workflow LAZY (Version du workflow figée pour conserver l'état historique)
User ManyToOne Role EAGER (Chargement obligatoire du rôle lors de l'auth)

Table : INCIDENT

Attribut Type SQL Description & Association
id (PK) BIGINT (Auto-increment) Clé primaire technique de l'incident.
incident_code VARCHAR(255) (Unique) Code séquentiel généré lors de la création (ex: INC-2026-xxx).
title VARCHAR(255) Titre descriptif concis de l'incident.
description TEXT Corps de description détaillé de l'anomalie.
priority VARCHAR(255) Niveaux : Low, Medium, High, Critical.
status VARCHAR(255) État actuel au sein du workflow de sa catégorie.
category VARCHAR(255) Catégories métiers : Réseau, Sécurité, Système, Médical.
author_id (FK) BIGINT Clé étrangère reliant le créateur (User).
assigned_to_id (FK) BIGINT Clé étrangère reliant l'opérateur en charge (User).
created_at TIMESTAMP Horodatage exact de déclaration.

Validation de session via Blacklist Redis

Pour assurer la révocation instantanée des sessions de Keycloak (Ex: déconnexion globale d'un opérateur), le filtre Java SessionValidationFilter intercepte chaque appel API et vérifie l'absence de l'email de l'appelant dans le cache Redis :

backend/src/main/.../config/SessionValidationFilter.java 
// Intercepte les requêtes REST pour valider la session
String email = extractEmailFromToken(token);
String redisKey = "blacklist:user:" + email;

if (redisTemplate.hasKey(redisKey)) {
    // Le token a été révoqué par Keycloak Webhook
    response.setStatus(HttpServletResponse.SC_UNAUTHORIZED);
    response.getWriter().write("{\"message\": \"Session révoquée.\"}");
    return;
}
filterChain.doFilter(request, response);
